# CAD 绘图复盘：坑清单与经验（FP-007 / FP-008 实战提炼）

本文档是"生成工程图为什么这么慢、如何变快"的复盘沉淀。SKILL.md 是流程，本文档是原理与教训。
目录：一、慢的根源与对策；二、COM 机制坑；三、图形/标注坑；四、验收方法论；五、开放问题。

## 一、慢的根源与改善对策（已并入规范）

| 慢的根源 | 对策 |
|---|---|
| 每次从零重写 COM 样板代码（连接/图层/文字样式/引线/图框/PNG 导出 ≈ 300 行） | 提炼 `scripts/acad_common.py` 公共库，新图只写几何与标注坐标 |
| 比例选定靠试错：画完发现标注挤不下 → 整图返工 | 比例评估前置（V4.3）：动笔前算"含标注总高 <150mm、净距 ≥15mm"，一次选对 |
| 引线/箭头方案不确定，反复试错（AddLeader 不显示箭头 → 改折线 → 又违反单段禁令） | 定案：单段直线 + AddSolid 实心三角箭头（probe_leader_arrow.py 实证），禁用 AddLeader |
| 形位公差框格用 AddTolerance，渲染挤压互相干涉，多次移位返工 | 定案：手工三格分置（外框矩形+竖分隔线+文字 MiddleCenter 逐格居中），禁用 AddTolerance |
| 标注干涉靠用户肉眼发现后再改 | 出图前几何避让验算（seg_pt_dist / seg_circle_cross 算净距）+ 局部放大 PNG 自查 |
| 字形缺字（Ø、↧）出图后才发现 | 定案：Φ=U+03A6、深度=↓ U+2193；新符号先用 _glyph_probe.py 探测再上图 |
| 绘图过程不可见，用户以为卡死 | 每实体 Update + 每阶段 Regen + 固定延时；只 sleep 不刷新 = 画面冻结（教训） |

## 二、COM 机制坑（acad_common.py 已封装，勿重写）

1. **RPC_REJECTED（-2147418111）**：AutoCAD 忙或弹对话框时 COM 呼叫被拒。call() 内置 sleep 1s 重试 120 次。
   持续被拒的排查：CAD 里是否有挂起的对话框、**挂起的文字在位编辑窗口**（遗留文档里双击过文字没退出）——
   可 PostMessage 向 AutoCAD 窗口发 ESC 解除。
2. **gencache 早期绑定缺属性**：部分属性/方法在早期绑定代理上缺失 → setp/getp 自动转 dynamic.Dispatch 回退。
3. **TrueColor**：图层设色必须 GetInterfaceObject("AutoCAD.AcCmColor.23")，直接设 Color 是 ACI 索引不是真彩色。
4. **遗留 Drawing\***：重跑脚本会不断累积未保存新文档 → new_drawing() 先 Close(False) 清理。
5. **PlotToFile 异步**：BACKGROUNDPLOT=0 前台打印同步返回；局部放大必须 **先 SetWindowToPlot 再设 PlotType=4**，顺序反了窗口不生效。
6. **SaveAs 禁令**：任何情况下不保存；审计比对磁盘 .dwg mtime + 新文档 FullName 必须为空字符串。

## 三、图形/标注坑

1. **DASHED2 偶发退化为实线**：图层线型设置后必须读回校验（make_layers 内置），加载用"存在性检查+双 .lin 源重试"。
2. **LTSCALE**：acadiso.lin 下 0.4；若被迫用 acad.lin（英制定义）需 ×25.4。ensure_linetypes 自动适配。
3. **同侧尺寸小内大外**：FP-007 曾把 100 放在 80 内侧被用户纠正——**已定案为永久规则**（规范 V1.6）。
4. **引注干涉**：Φ60 引注曾与相邻尺寸线干涉 → 规则：引注互不干涉，过近就移位/旋转到空旷方向；倒角引注贴近倒角。
5. **引线不得折弯**（V1.6 禁令）：曾经为绕开图形折弯引线，被用户否决——宁可移动文字位置。
6. **第一角投影**：中国国标第一角法——左视图放右边、俯视图放下边。侧视图方向极易画反，动笔前在 docstring 写清"从哪面看、凸台朝哪边"。
7. **螺纹端面表达惯例**：小径粗实线（或虚线）整圆 + 大径细线约 3/4 圆（45°→315°）。

## 四、验收方法论（不过夜原则：当轮举证当轮看清）

1. COM 自审计断言（实体计数、关键文字逐条、无 PCD、线型读回、引线单段、框格居中、mtime 不变）——脚本尾部 assert，不过即失败。
2. 整图横向 PNG（monochrome.ctb）看整体布局。
3. **局部放大 PNG**（每处易错区一张：引注区、框格、偏差堆叠）——必须亲自读图验收，不能只看审计文本。
4. 汇报必含：用户指令与规范默认的差异清单（如"（均布）"句式、五层中文层名、侧视图不剖）。
